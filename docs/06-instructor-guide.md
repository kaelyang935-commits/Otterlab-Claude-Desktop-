## 2. 建議授課順序（為什麼這樣排）

```
⓪ 安裝 + 第一個成果（第0課）
   → 先讓每個人都能登入、進到 Code 分頁、用一句話做出一個網頁,建立信心
① 基礎概念（第1課）+ 安全三底線
   → 建立「指令 vs 聊天」心智模型;同時把三條安全底線講清楚(別等到第9課才講)
②  /help + /powerup（讓學員自己探索）
   → 互動感強，學員馬上有參與
③ 對話管理四天王：/clear /compact /rewind /resume
   → 每天都會用的，先練到反射
④ 程式碼工作流：/init → /code-review → /security-review（開發者班）
   → 串成一條真實的開發流程示範;非工程師班可略過
⑤ 自訂指令（第4課,先教「30 秒純文字版」）
   → 從「使用者」升級成「能擴充工具的人」;新手用無 git 的 /polish 練手,成就感最高
⑥ 環境差異（第5課）
   → 講四環境互通與跨裝置接續
⑦ 技能哲學（第7課）：官方先用、別替換
⑧ 缺的去 GitHub 找一份 .md（第8課）
⑨ 動手做:貼 URL 叫 agent 一鍵執行（第9課,新手成就感最高,適合當收尾實作）
```

> 🎯 **教學心法：** 不要一開始就把 15 大類全倒給學員。**先用第 0 課做出一個成果、再給「快速開始 5 個」建立信心，最後才分類補完。**

## 3. ⚠️ 學員最常見的 6 個誤區（務必課堂點破）

| # | 誤區 | 正解 |
|---|---|---|
| 1 | 把 `/` 打在句子中間還期待它執行 | 指令**只在訊息開頭**才生效 |
| 2 | 以為 `/clear` 跟 `/compact` 一樣 | `/clear`＝清空開新；`/compact`＝摘要壓縮（內容還在） |
| 3 | 改壞東西想救回卻用 `/clear` | 要用 `/rewind`（倒回某點，含程式碼） |
| 4 | 分不清 `/review` 和 `/code-review` | `/review`＝GitHub PR；`/code-review`＝本機 diff |
| 5 | 以為 `/model` 和 `/effort` 是同一件事 | 換模型 vs 同模型思考多深 |
| 6 | 自訂指令放錯位置，團隊看不到 | 團隊共用要放**專案** `.claude/` 並進版控 |

## 4. 課堂示範腳本（講師唸稿範例）

> 以下是「對話管理四天王」段落的逐字示範稿，可直接照念。

**【示範 /context 與 /compact】**

> 「各位看，我這段對話已經聊很久了。我先打 `/context`——大家看到這個彩色格狀圖了嗎？它告訴我目前用掉多少『記憶空間』。當它快滿，Claude 會變慢、也比較花錢。
> 這時我不想全部清掉、又想省空間，就打 `/compact`。看，它把前面的對話『摘要壓縮』了，重點都還在，但占用空間變小。
> 注意喔，這跟 `/clear` 不一樣——`/clear` 是整個倒掉重來。」

**【示範 /rewind】**

> 「假設我剛剛讓 Claude 改了三個檔案，結果改壞了。新手會想『那我 /clear 重來』——錯！`/clear` 救不回程式碼。
> 正確做法是 `/rewind`，它能把『對話』甚至『程式碼』倒回剛才那個還沒壞掉的點。這是你的後悔藥。」

## 5. 隨堂評量題（附解答）

**選擇題**

1. 下列何者**不會**被當成指令執行？
   - (A) `/usage` (B) `請 /usage` (C) `/usage 一下`
   - <details><summary>解答</summary>(B)。`/` 不在開頭。</details>

2. 想把對話「壓縮但保留重點」，用？
   - (A) `/clear` (B) `/compact` (C) `/rewind`
   - <details><summary>解答</summary>(B)</details>

3. 審查「本機尚未提交的程式碼變更」用？
   - (A) `/review` (B) `/code-review` (C) `/diff`
   - <details><summary>解答</summary>(B)。`/review` 是 GitHub PR。</details>

**問答題**

4. 用自己的話解釋「內建指令」與「技能指令」的差別。
   - <details><summary>參考</summary>內建＝一個直接動作（如開關）；技能＝一整套工作流程，Claude 會自己用工具完成（如呼叫專家）。</details>

5. 你要做一個團隊共用的 `/deploy` 指令，檔案放哪？為什麼？
   - <details><summary>參考</summary>放專案 `.claude/commands/`（或 `.claude/skills/`）並進 git，團隊 clone 即可共用。</details>
