## 1. 什麼是斜線指令（Slash Command）？

斜線指令是**以 `/` 開頭的特殊指令**，用來「指揮」Claude Desktop 本身做事——例如清空對話、切換模型、審查程式碼、查看花費等。

它跟你平常打的「自然語言訊息」不同：

| | 自然語言訊息 | 斜線指令 |
|---|---|---|
| 例子 | 「幫我把這段程式碼重構」 | `/clear`、`/model`、`/compact` |
| 對象 | 交給 AI 思考、回答 | 由程式直接執行某個動作 |
| 開頭 | 任意 | **一定是 `/`** |

> 🧠 **比喻：** 自然語言是「跟 AI 聊天」，斜線指令是「按下 App 上的功能按鈕」，只是用打字的方式按。

## 2. 四個運作規則（最重要！）

### 規則 ①：指令一定要放在訊息「開頭」

✅ 正確：
```
/compact 只保留 API 相關的討論
```

❌ 錯誤（`/` 不在開頭，會被當成普通文字）：
```
請幫我 /compact
```

### 規則 ②：指令名稱後面接的文字 = 參數

```
/model opus          ← opus 是參數（要切換的模型）
/clear 上午的需求討論   ← 後面整段都是參數（替舊對話命名）
```

### 規則 ③：打一個 `/` 會跳出指令選單

在輸入框打 `/`，會出現可用指令清單；**繼續打字會即時篩選**。例如打 `/co` 會篩出 `/compact`、`/context`、`/config`、`/copy`…

### 規則 ④：有些指令會開「互動面板」

像 `/config`、`/permissions`、`/agents`、`/hooks`、`/mcp` 會跳出互動式設定畫面，需要終端機或桌面 App 的介面才能完整操作（網頁版受限）。詳見 [第 5 課](05-environments.md)。

## 3. 三大類指令

學員最常見的困惑是「為什麼有些指令長得不一樣？」——因為它們其實分成三類：

### 🔵 第一類：內建指令（Built-in）

由程式本身提供，控制工作階段、設定與工具。
**例：** `/clear`、`/model`、`/help`、`/config`、`/usage`

➡️ 完整清單見 [第 2 課](02-builtin-commands.md)。

### 🟢 第二類：技能指令（Skills）

用「提示詞」驅動的工作流程：你下指令，Claude 會依內建的詳細步驟去執行一整套任務，過程中會自己使用各種工具。
**例：** `/code-review`、`/security-review`、`/deep-research`、`/loop`、`/run`

➡️ 詳見 [第 3 課](03-skills.md)。

### 🟣 第三類：自訂指令（Custom）

你自己或團隊寫的 `.md` 檔，做成專屬指令。
**例：** 你可以做一個 `/commit` 自動產生 git commit 訊息、`/standup` 產生每日進度報告。

➡️ 動手教學見 [第 4 課](04-custom-commands.md)。

| 類別 | 誰提供 | 能不能改 | 範例 |
|---|---|---|---|
| 🔵 內建 | Claude Code 本身 | ❌ | `/clear`、`/model` |
| 🟢 技能 | 內建或外掛 | ⭕（可加裝） | `/code-review` |
| 🟣 自訂 | 你 / 你的團隊 | ✅ 完全自由 | `/commit`（你寫的） |

## 📝 課堂練習

> 請在 Claude Desktop 輸入框實際操作，並把觀察到的結果記下來。

1. **練習 1（選單）：** 在輸入框打一個 `/`，看看跳出多少指令？再打 `/co`，剩下哪些？
2. **練習 2（說明）：** 執行 `/help`，找出 3 個你之前沒看過的指令，寫下它們的用途。
3. **練習 3（參數）：** 執行 `/clear 我的第一次練習`，然後執行 `/resume`，你能在清單中找到剛剛命名的對話嗎？
4. **思考題：** 下面哪一行**不會**被當成指令執行？為什麼？
   - (A) `/usage`
   - (B) `麻煩 /usage 一下`
   - (C) `/compact 保留重點`

<details>
<summary>👉 點此看練習 4 解答</summary>

答案是 **(B)**。因為 `/` 沒有在訊息**開頭**（規則 ①），所以整句會被當成普通的自然語言訊息送給 AI，而不會執行指令。

</details>

## 🔑 重點回顧

- 斜線指令 = 用打字的方式「按 App 的功能按鈕」。
- 四規則：**開頭**、**後接參數**、**`/` 叫出選單**、**部分是互動面板**。
- 三類別：🔵 內建、🟢 技能、🟣 自訂。

---

## 🛡️ 安全底線（每一課都適用,請記住）

1. 要 AI 跑「別人的程式或技能」前,先請它「**只摘要、先別執行**」。
2. 不確定的網址或檔案,**不要叫它「照做」**。
3. 看到要**刪檔、部署、付費、寄信**這類動作,先停下來確認。

> 這三條在 [第 9 課](09-run-skill-from-url.md) 會解釋背後的原理(提示注入)。但**不必等到第 9 課**——從今天就養成習慣。
