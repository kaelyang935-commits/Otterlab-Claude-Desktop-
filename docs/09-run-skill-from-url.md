# 第 9 課（動手做）｜🟢 給 agent 一個 URL,一鍵執行技能(零安裝)

> 上一課：[第 8 課 找到適合你任務的 Skill .md](08-find-skill-md.md) ｜ 回到：[課程首頁](../README.md)

> 本法最適合 **Claude Desktop / Claude Code 的 Cowork 模式**(agent 能讀網頁、也能用工具動手做)。
> 純聊天介面也能讀網址,但要「實際執行任務」需另外開啟網頁存取與程式執行。

---

## 🎯 你會做到

**不安裝任何技能**,只給 agent 一個 GitHub `.md` 網址,就讓它照那份說明完成任務。

---

## 為什麼用這個方法?

- **零安裝、零設定**,新手 30 秒上手。
- 適合「只想用一次」或「先試試看這個技能好不好用」。
- 配合 [第 8 課](08-find-skill-md.md):**找到一份適合的 `.md` → 貼網址叫它做**,就是最輕量的完整流程。

---

## ✅ 前置條件

- 用 **Claude Desktop(Cowork 模式)或 Claude Code**。
- agent 具備**讀網頁**的能力(內建)。要它實際**跑程式**(如合併 PDF),需有對應工具/程式執行可用。
- 目標 `.md` 必須在**公開** repo(私有的要另外授權)。

---

## 步驟 1️⃣ 找到技能的 `.md`

照 [第 8 課](08-find-skill-md.md) 找到一份適合你任務的技能,打開它的 `SKILL.md`(或任何當說明的 `.md`)。

> 例:官方 PDF 技能 → 開 [github.com/anthropics/skills](https://github.com/anthropics/skills) → 進 `skills/pdf/SKILL.md`

## 步驟 2️⃣ 拿到網址(raw 最穩)

在該檔頁面點 **Raw** 鈕,複製網址列,會長這樣:

```
https://raw.githubusercontent.com/anthropics/skills/main/skills/pdf/SKILL.md
```

> 直接複製 github.com 的 blob 網址(`.../blob/...`)通常也行,但 raw 連結最穩。

## 步驟 3️⃣ 貼給 agent + 一句話

把網址貼進 Claude,前面加一句「讀這個並照做」:

> 讀取這個網址的內容,並依照裡面的說明完成任務:
> `https://raw.githubusercontent.com/anthropics/skills/main/skills/pdf/SKILL.md`
> 我要把 `a.pdf` 和 `b.pdf` 合併成一個 `merged.pdf`。

## 步驟 4️⃣ 看 agent 執行

agent 會:① 讀取那份說明 → ② 理解該技能的做法 → ③ 用工具完成你的任務。

---

## 💬 示範對話(照這個感覺進行)

> 以下為示意,實際輸出依你的檔案與環境而定。

**👤 你：**
> 讀取這個網址的內容,並依照裡面的說明完成任務:
> https://raw.githubusercontent.com/anthropics/skills/main/skills/pdf/SKILL.md
> 我要把 a.pdf 和 b.pdf 合併成一個 merged.pdf。

**🤖 Claude：**
> (正在讀取網址…)我已讀完這份 PDF 技能說明,它建議用 Python 的 `pypdf` 來處理。我來把 `a.pdf`、`b.pdf` 依序合併:
>
> ```python
> from pypdf import PdfWriter
> writer = PdfWriter()
> for f in ["a.pdf", "b.pdf"]:
>     writer.append(f)
> writer.write("merged.pdf")
> ```
>
> ✅ 完成!已輸出 `merged.pdf`(共 12 頁)。需要我再幫你加頁碼或浮水印嗎?

**👤 你：**
> 太好了,順便幫我在每頁右下角加頁碼。

**🤖 Claude：**
> 沒問題,我依同一份技能說明的「進階」段落處理…(繼續執行)

> 🔎 **看懂了嗎?** 你沒有安裝任何東西——agent 只是「讀了一份網路上的說明書」,然後照著做。這就是最輕量的技能調用法。

---

## 🚀 進階用法

- **先審再做(最推薦):** 「**先幫我摘要這份技能會做什麼**,我同意再執行。」→ 安全又清楚。
- **結合你的資料:** 把網址 + 你的檔案/需求一起給,讓它針對你的情況做。
- **一次多份:** 可貼多個 `.md` 網址,請它整合運用。

---

## 🧰 疑難排解

| 症狀 | 解法 |
|---|---|
| agent 說讀不到網址 | 確認是**公開** repo;改用 **raw** 連結;確認網址沒打錯 |
| 讀到一堆 HTML 雜訊 | 你給的是網頁版(blob)網址,改用 **Raw** 連結 |
| 讀到了卻沒動手做 | 明確說「**並依照裡面的說明完成任務**」,並補上你的具體需求 |
| 要它跑程式卻失敗 | 該環境未啟用程式執行/對應工具;改用 Cowork 模式或開啟程式執行 |
| 私有 repo 讀不到 | 私有內容需另外授權;或把該 `.md` 內容直接貼給 agent |

---

## 🔒 安全提醒(這課最重要)

> **把網址叫 agent「照做」,等於讓「遠端文件」直接指揮你的 agent ——它可能據此跑指令、改檔案、存取你的資料。這本質上是一種「設計上的提示注入」。**

- **只指向你信任的網址**(官方 `anthropics/*` 最安全)。
- **最好先讀過那份 `.md`**,或先請 agent「**先摘要再執行**」。
- 對陌生來源,別直接叫它「照做」;先看清楚它要做什麼。
- 網址要看清楚網域,小心被偽裝的連結。

---

## 📝 課堂練習

1. **實作:** 用步驟 2 的 raw 連結,叫 agent「**先摘要這份 PDF 技能會做什麼**」(先別執行)。
2. **對照:** 「做一份 PPT(用官方技能)」與「給一份 `.md` 的 URL 用外部技能」,分別適合什麼情境?
3. **安全題:** 同學傳你一個陌生的 GitHub `.md`,叫你貼給 agent 執行。你會先做什麼?
4. **思考題:** 什麼情況該把這份 `.md` **存進 `~/.claude/skills/`** 長期使用,而不是每次貼 URL?

<details>
<summary>👉 點此看參考答案</summary>

2. 日常、官方就有的(做 PPT)→ 直接用官方技能;官方沒有、臨時要用的特定任務 → 找一份 `.md` 給 URL 最快。
3. 先**打開那份 `.md` 自己讀**,或請 agent「先摘要不要執行」,確認它做什麼、來源可信,再決定。
4. 當你**會重複使用同一個技能**時——存進 `~/.claude/skills/<名稱>/` 後打 `/<名稱>` 就能用,不必每次貼 URL。

---

## 🔑 重點回顧

1. 找到技能的 `.md` → 點 **Raw** 拿網址 → 貼給 agent 說「**讀這個並照做**」+ 你的需求。
2. 零安裝,最適合新手與「用一次」的情境。
3. ⚠️ 等於讓遠端文件指揮 agent:**只用可信網址,最好先讀過或先請它摘要再執行。**
4. 會重複使用,就把那份 `.md` 存進 `~/.claude/skills/<名稱>/` 長期用。
